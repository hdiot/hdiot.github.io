嵌入式课程设计报告

班级：物联1151     姓名：聂兆炜     学号：2015**

Linux 内核移植

初步配置

  • 在系统中添加对ARM的支持：有两种方法

    • 第一种：修改顶层 Makefile 文件
      // 原来的代码
ARCH ?= $(SUBARCH)
CROSS_COMPILE ?= $(CONFIG_CROSS_COMPILE:"%"=%)

// 修改为
ARCH ?= arm
CROSS_COMPILE ?= arm-linux-
    • 第二种：编译时使用参数设置
      // 使用此命令配置kernel
# make ARCH = arm CROSS_COMPILE = arm-linux- menuconfig

  • 使用 s5pv210 的默认配置文件生成默认的 .config 配置文件

    // 命令 s5pv210_defconfig为配置文件名
# make s5pv210_defconfig

  • 生成配置文件后执行命令

    # make menuconfig
    进行初步配置
    • System Type –>
      • 根据开发板设置调试串口
        (0) S3C UART to use for low-level message
      • 去掉其他CPU（S5PC110）的支持
        S5PC110 -->
    [] Aquila
    [] GONI
    [] SMSKC110
      • 保留 S5PV210 Machines –>
        [*] SMDKV210
[]  Torbreck
      • 保存并退出 完成初步配置

  • 对内核进行编译

    # make zImage
// 或者
# make uImage
// 具体使用哪一种根据uboot决定

    编译完成，生成 uImage 文件（修改文件名为 uImage.bin）

  • 将 uImage 文件 使用 TFTP 下载到 内存中

    注意：地址在内存允许范围内任意，尽量靠前到0x20000000

    • uImage将文件拷贝到 TFTP 服务器目录

    • 使用SecureCRT烧录

      //烧录     地址     文件名（注意添加.bin后缀）
# tftpboot 20000000 uImage.bin 

// 启动内核
# bootm 20000000

网卡移植

相关参考帮助文档：Documenttation/networking/dm9000.txt

  • mach-smdkv210.c（arch/arm/mach-s5pv210目录下） 配置了DM9000 的平台设备的相关数据

    所以只需要进行适当修改

  • 定义 S5PV210_PA_SROM_BANK1 的基地址

    arch/arm/mach-s5pv210/include/mach/map.h 定义了S5PV210_PA_SROM_BANK5的基本地址

    # define S5PV210_PA_SROM_BANK1  0x88000000
# define S5PV210_PA_SROM_BANK5  0xA8000000

  • 修改 mach-smdkv210.c 中DM9000 配置

    // 原来的代码
static struct resource smdkv210_dm9000_resources[] = {
    [0] = DEFINE_RES_MEM(S5PV210_PA_SROM_BANK5,1),
    [1] = DEFINE_RES_MEM(S5PV210_PA_SROM_BANK5,1),
    [2] = DEFINE_RES_NAMED(IRQ_EINT(9),1,NULL,IORESOURCE_IRQ \
                            | IORESOURCE_IRQ_HIGHLEVEL),
}

// 修改后
static struct resource smdkv210_dm9000_resources[] = {
    [0] = DEFINE_RES_MEM(S5PV210_PA_SROM_BANK1,4),
    [1] = DEFINE_RES_MEM(S5PV210_PA_SROM_BANK1,4),
    [2] = DEFINE_RES_NAMED(IRQ_EINT(10),1,NULL,IORESOURCE_IRQ \
                            | IORESOURCE_IRQ_HIGHLEVEL),
}
    TQ210 开发板使用的DM9000 使用的外部中断 10
* 屏蔽SROM初始化函数 smdkv210_dm9000_init()

        static void __init smdkv210_dm9000_init(void){
            // 将函数调用注释
            // smdkv210_dm9000_init();
        }

  • 配置内核支持网络文件系统（NFS）

    内核未移植NAND FLASH，开发板只能使用网络文件系统启动，需配置内核支持网络

    [*] NetWorking support  -->
    NetWorking option   -->


    Device Drivers -->
    [*] Ethernet driver support(NEW) -->


    File System --> 
    [*] Network File System(NEW) -->
        <*> NFS client support
        [*] Root file system on NFS

  • 编译内核

  • 制作根文件系统

    需要从NFS启动开发板

    • 使用BusyBox制作根文件系统

      • 配置：

        General Configuration -->
    [] Enable options for full-blown desktop systems
    // ps 命令显示进程状态
Build Options -->
    （arm-linux-）Cross Compiler prefix
    // 指定交叉编译前缀
Installation Options ("make install" behavior) -->
    (.install) BusyBox installation prefix (NEW)
    // 指定安装路径
Init Utilities -->
    [] Be_extra_quiet on boot
    // 取消系统启动显示BusyBox版本号加载和卸载内核模块
Linux Module Utilities -->
    [*] insmod
    [*] rmmod
    [*] Ismod
    [*] modprode
    [*] depmod
    // 是适应简单 使用insmod工具
      • 编译和安装
        # make && make install
      • 将编译安装好的文件系统复制到指定目录
      • 修改文件系统的环境变量
      • 开启服务主机的 NFS 服务

      • 设置uboot NFS启动参数

        noinitrd console=ttySAC0 root=<主机IP>:/nfsroot/rootfs rw ip=<开发板IP>:<主机IP>:<网关>:<子网掩码>::eth0:off init=/linuxrc
      • 重启NFS服务

总结

回首坎坷历程：

从安装和配置嵌入式开发环境，就掉了不少的坑，一会是驱动不行、一会gcc编译失败，重点是还不知道问题出在哪里，一路走来慢慢的都是泪，还把队友不知不觉的带进坑里。遇到问题，只能一个个Google了。

收获：

遇到问题，证明自己的知识不足；解决问题，证明自己在成长。整个过程下来，虽然没有成功的移植内核和文件系统，但自己对系统、内核有了更深入的了解。
