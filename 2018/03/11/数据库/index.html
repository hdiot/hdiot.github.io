##数据库简介

  • 数据库出现原因：
    • 数据量增大带来的数据记录和查找问题
    • 计算机的诞生
  • 数据库要结局的问题：
    • 持久化存储
    • 优化读写
    • 保证数据有效性
  • 数据可分类：
    • 文档型：SQLite
    • 服务型：MySQL、Oracle
      • 组成：
        • 服务端
        • 客户端

E-R模型

  • 当前物理的数据库都是按照E-R模型进行设计的
  • E entry 实体
  • R relationship 关系
  • 一个实体转换为数据库的表
  • 关系描述两个实体之间的对应规则，包括：
    • 一对一
    • 一对多
    • 多对多
  • 关系转换为数据库表中的一个列 关系数据库中一个列就表示一个对象

三范式

  • 范式：为了解决数据可使用中的问题，在数据库设计时的规范–范式
  • 范式分类：
    • 第一范式 1NF：列不可拆分
    • 第二范式 2NF：唯一标识
    • 第三范式 3NF：引用主键
  • 后一个范式都是前一个范式的基础是建立的

查询

简介

  • 基本语法

    select * from 表名;

  • from关键字后面写表名表示的数据来源于这张表

  • select后面写表中的字段名，如果是*表示在结果中显示所有字段
  • 在select后面的字段名，可以使用as为字段起别名，此别名出现在结果集中
  • 如果要查询多个字段，之间使用逗号隔开

消除重复的行

  • 在select后面字段名前面使用distinct可以消除重复的行

    select distinct gender from students;

条件

  • 使用where条件语句对表中数据筛选，结果为true的行会出现在结果集中

  • 语法：

    select * from 表名 where 条件;

比较运算符

  • 等于：=
  • 大于：>
  • 大于等于：>=
  • 小于：<
  • 小于等于：<=
  • 不等于：!= 或者 <>

  • 例子：

    # 查询编号大于3的学生
select * from students where id > 3;

# 查询编号不大于4的学生
select * from subjects where id<=4;

聚合

  • 快速得到统计数据，5个聚合函数

  • count() 表示统计总行数，括号内写或列名，结果相同

    # 查询学生总数
select count(*) from students；

  • max(字段名) 表示此列的最大值

    # 查询女生的编号的最大值
select max(id) from students where gender =0;

  • min(字段名) 与 max()相反

    # 查询未删除的学生的最小编号
select min(id) from students where isdelete=0;

分组

  • 按照字段分组，表示此字段相同的数据会被收到一个组中
  • 分组后只查询出相同的数据列，对于差异的数据列无法出现在结果集中
  • 可对分组后的数据进行统计，做聚合运算

  • 语法：

    select 列1, 列2，聚合... from 表名 group by 列1, 列2，列3...;

  • 例子：

    # 查询男女生总数
select gender as 性别,count(*)
from students 
group by gender;

# 查询个城市总人数
select hometoen as 家乡,count(*)
from studentsgroup by hometown;

  • 分组后的数据筛选

    • 语法：

      select 列1,列2,聚合... from 表名
group by 列1,列2,列3...
having 列1,...聚合...

    • having 后面的条件运算符与where的相同

    • 例子：

      方案一
select count(*)
from students
where gender=1;
-----------------------------------
方案二：
select gender as 性别,count(*)
from students
group by gender
having gender=1;
